## Scene7 IPS (Image Production System) 4.5.0 {#section-3e37ed37b6d146419f4578bd7ffeaa74}

* Ora in esecuzione attraverso JDK 1.8.
* Supporto per il tag di orientamento EXIF.
* Carica file di directory salta per file FTP temporanei (come creato con ProFTP).

**Funzioni non più supportate**

* Opzione di risincronizzazione del server di catalogo all’interno di Convalida pubblicazione IPS Classic - rimossa.
* Non è più possibile impostare valori di campo dei tag di metadati vuoti o duplicati.

**Correzioni di bug**

* La `<metadataArray>` era vuoto per [!DNL .vtt] file in risposta all’API `getAssets`.

* Campo dati utente mancante per l&#39;orientamento.
* Risposta da `getJobLogDetails` per un processo che aggiorna un file ACO non specificava correttamente i dettagli del processo.
* Il file primario di PSD è stato registrato come pubblicato più volte per Image Server.
* È possibile caricare in IPS le risorse il cui nome contiene caratteri non validi.
* Le miniature video che non è stato possibile generare sono state lasciate in uno stato &quot;In attesa in coda&quot;.
* Race condizione con ripristino della cartella tra grandi dimensioni `deleteFolder` e il caricamento della directory del server.
* I file PDF venivano registrati come pubblicati due volte per server di pubblicazione quando venivano pubblicati all&#39;inizio del caricamento.
* Il caricamento di più file PSD con gli stessi nomi di livelli ha causato errori di ottimizzazione.
* La rasterizzazione PDFL di customer PDF ha generato una linea bianca inaspettata nell’immagine generata.
* I file delle miniature video sono stati pubblicati più volte al momento del caricamento.
* Il rilevamento dei file di caricamento duplicati ha generato due notifiche di caricamento, eliminando la contabilità dei file completata.
* Il file XML Sidecar per la transcodifica video è stato registrato come pubblicato più volte al momento dell’aggiornamento.
* moveFolder non è riuscito quando la cartella nidificata è stata eliminata e un&#39;altra cartella, che non si trova nella cartella padre della cartella nidificata e ha lo stesso nome della cartella nidificata, è stata spostata nella posizione in cui si trovava la cartella nidificata.
* Le voci di configurazione dello stato del cluster nella configurazione condivisa e locale non sono state eliminate tramite la pulizia.
* Modifica il parser di registro API per filtrare i formati di indirizzo e-mail.
* L&#39;elaborazione video non viene mappata `Iptc4xmpCore:Location` correttamente.
* Tra la pulizia e il caricamento esisteva una race condition.
* Pubblicazione non utilizzata `assetFile` la pubblicazione in staging funziona solo per le nuove risorse.
* Puntatore Null generato durante l&#39;aggiornamento di un set di regole.

## Visualizzatori {#section-833ab92c91c941d2bfdc27f233f582ad}

**Nuove funzioni, correzioni e miglioramenti implementati in Dynamic Media Image Serving Viewers 5.2.2**

Gli aggiornamenti del visualizzatore sono compatibili con le versioni precedenti: non sono necessarie modifiche al codice web esistente. Tuttavia, si consiglia di testare i nuovi visualizzatori nell’ambiente di staging di Adobe. Il seguente sito web fornisce istruzioni su come configurare il sistema per accedere al server di staging di Adobe:

[https://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html](https://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html)

Dopo aver configurato il computer per accedere al server di staging, è possibile controllare il sito web per verificare l&#39;aggiornamento. Se utilizzi visualizzatori predefiniti di HTML5, la best practice prevede di eseguire il test rispetto al server di staging standard di Adobe e a s7is5-preview-staging.scene7.com.

Nuove funzioni e miglioramenti

Vedi anche [Guida di riferimento visualizzatori](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/homeviewers.html?lang=en).

* È stato aggiunto il supporto per Stampa, Download e Preferiti nel visualizzatore eCatalog.
* È stata aggiunta la possibilità di recuperare ParameterManager utilizzando l’API getComponent.
* Visualizzatore 360 gradi, visualizzatore zoom, visualizzatore video e visualizzatore a comparsa convertiti per utilizzare gli spriti per le immagini.
* È stato aggiunto il supporto per IE11 a schermo intero nativo.
* Supporto a schermo intero simulato (non nativo) modificato in Container.
* È stata aumentata la dimensione dei marker CSS per supportare i telefoni più grandi.
* Sono stati rimossi i marcatori CSS per i visualizzatori HTML5 per set a 360 gradi, BasicZoom, Zoom, Spin e MixedMedia sui browser desktop.
* È stato aggiunto il supporto per consentire la configurazione di qualità dei fotogrammi precaricati in SpinView.
* Problema corretto - Galaxy S4, modalità verticale: Indicatore di dimensione CSS errato utilizzato quando è stata abilitata la modalità a schermo intero.
* Fisso [eCatalog] Le mappe immagine sono bloccate dopo aver utilizzato modelli JavaScript per il collegamento esterno su Internet Explorer 9.
* Problema corretto: Internet Explorer 9 e Internet Explorer 10: lo zoom funziona in modo errato se la proprietà larghezza è definita per gli IMG in CSS.

* È stato risolto un problema a causa del quale s7sdk.browser.device.version ha rilevato il quinto Android™ in modo errato.

Visualizzazione zoom a comparsa

* Fisso [IE9] Il riquadro a comparsa a volte è rimasto visualizzato dopo lo spostamento del mouse.
* Problema corretto: la stringa iscommand veniva aggiunta due volte all&#39;immagine iniziale.

PageView

* Problema corretto: la rotazione della pagina non funzionava correttamente se il movimento di trascinamento iniziava verticalmente e quindi veniva spostata orizzontalmente.
* Sono stati aggiunti altri eventi di stato.

SpinView

* Implementazione della rotazione automatica di HTML5.
* Sono stati aggiunti altri eventi di stato.

Campioni

## Funzioni non più supportate {#section-9295ea9313c84695b19bca0e0f01a690}

* **Avviso sulla fine del ciclo di vita dei visualizzatori di Flash** *Dal 31 gennaio 2017, Adobe Scene7 (ora Dynamic Media Classic) ha ufficialmente terminato il supporto per la piattaforma di visualizzatori di Flash.*

* Promemoria: A partire dal 31 gennaio 2014, Scene7 ha ufficialmente terminato il supporto per la piattaforma di visualizzatori DHTML.
* Le estensioni di Scene7 Creative Suite sono obsolete in questa versione a causa del supporto dei pannelli di Flash da parte dell’applicazione Creative Cloud. Ulteriori sviluppi dei flussi di lavoro per l’acquisizione di contenuti sono incentrati su Adobe Experience Manager Dynamic Media e Creative Cloud. Per Scene7, le funzionalità di caricamento ed esportazione in Scene7 Publishing System rimangono e possono essere utilizzate al loro posto.
* I video UGC sono diventati obsoleti in questa versione a causa di una bassa adozione. La configurazione di un altro account Dynamic Media Classic può fungere da soluzione sostitutiva per i video UGC. Per configurare un altro account, contatta il supporto tecnico Adobe.
* La sillabazione del testo in questa versione è obsoleta a causa della bassa adozione. I dizionari di sillabazione definivano le modalità di suddivisione e sillabazione delle parole per ottimizzare il flusso del testo.
* Il server metadati e la ricerca per similarità sono stati entrambi obsoleti nel 2015 a causa della bassa adozione. Quando viene annunciata la data di scadenza ufficiale, i clienti interessati sono stati informati dal supporto tecnico Adobe.
